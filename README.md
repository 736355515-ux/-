## 一、赛题背景

无线网络的话务时空分布高度动态化，给运营商带来两类挑战，本赛题据此拆成两道题：

### 赛题一：AI 话务信息预测

- **潮汐效应显著**：商务区（CBD）工作日白天话务量可达夜间的 10–50 倍，住宅区呈相反规律。
- **突发场景难预测**：演唱会、赛事、交通事故等大型活动会导致话务热点迅速涌现。
- **节能与体验的矛盾**：基站长期"常开"能耗占运营成本的 20–30%，盲目关断又会引发拥塞。

因此，**准确预测未来 24 小时的逐小时话务分布**成为预测性节能与运维优化的关键输入。

- **输入**：历史连续 14 天的小时级话务量、站点工参、天气数据。
- **输出**：未来 24 小时、每小时 4 个话务指标（上/下行平均激活用户数、上/下行平均 PRB 数）。
- **评价指标 MAPEAUC**：对每个样本时刻计算 4 指标的平均绝对百分比误差 MAPE，
  统计 `MAPE < 0.2 / 0.3 / 0.4 / 0.5` 四个阈值下的样本占比，各占 25% 加权：

  ```
  MAPEAUC = 0.25·R(0.2) + 0.25·R(0.3) + 0.25·R(0.4) + 0.25·R(0.5)
  ```

  判题前对样本做两阶段过滤：剔除真实值为空（NIL）的时刻，并按每个测试集独立计算各指标
  真实值的 5% 分位数、剔除极端低值时刻。

### 赛题二：通信资源联合分配

面向无线多天线全连接场景，基站通过 TRX 通道发送波束，波束如同"探照灯"服务不同用户。
需要在给定带宽资源、波束能力上限与用户请求下，**联合决策"资源块子带 → 波束"与"用户 → 资源块"
的分配，最大化所有用户成功传输的数据总量 T**。

- 用户 `buffer` 为待传数据量，`cap[u][beam]` 为用户在各波束上的传输能力。
- 资源块集合 RES 被划分为若干子带 RSub（**决赛新特性：同一 res 可属于两个子带**）；
  每个子带可从波束集合中选取若干波束，全局波束用量受 `beamMaxNum` 上限约束。
- 用户集合含 **MU（可共享同一资源块的用户组 RU）** 与 **SU（不可共享）** 两类约束。
- 单用户在资源块上的等效信道 `fse` 由基本能力 SINR、共享人数惩罚、波束增益等按对数域公式合成，
  再经速率阶梯表映射为实际速率；`tranBuffer = min(buffer, bufCap)`，目标 `Maximize T = Σ tranBuffer`。
- **规模上限**：N≤100、K≤72、T≤18、P≤32、|RU|≤20、beamMaxNum≤255；**运行时限 100ms**。

这是一个带耦合约束的 NP-hard 组合优化问题，需在 100ms 内给出高质量可行解。

## 二、实现思路

### 赛题一：log 域异构集成 + 训练集留出验证的增量改进

最终模型为 **log 域的异构模型集成**，全部只在训练集上完成，严格不使用测试集话务：

| 成员 | 数量 | 说明 |
| --- | --- | --- |
| 日序列卷积网络 | 4 个 | 捕捉 14 天周期与日内形态 |
| 两阶段 LightGBM | 6 个 | 先拟合 log 话务，再用 MAPEAUC 阶梯的平滑代理损失微调 |
| MLP | 2 个 | 补充非线性 |

- **集成权重** 0.6 / 0.3 / 0.1（Conv / LGBM / MLP）。
- **数据预处理**：剔除非正值、NIL 空值，以及各指标训练分布的低 5% 尾部样本，对齐判题过滤口径。
- **决赛增量改动**（仅由训练集留出验证选定，防止过拟合线上）：
  1. 用 `seed=314` 的手写 attention 成员替换权重 0.15 的 `conv107` 成员，强度 1.2525；
  2. 第 3 个指标以 0.0215 的 log 域权重融合专家卷积成员。
- **合规复现**：训练/推理源码全部随包提交，随机种子显式固定
  （LightGBM `7/99/1907/4099/777/2718`，卷积 `7/107/2027/2718`，MLP `7/107`，增量成员 `314`），
  数据划分用 `RandomState(0)` 抽 25% 小区留出，逐条命令可复现。
- **测试集独立推理**：1849 个测试组、每 336 行为一组，每组只读本组 14 天话务，
  不跨组聚合、不做同名小区历史补全、不计算输入指纹、不查表，最终产出 44,376 行预测。

### 赛题二：单线程 SA + 贪心修复 + LNS 的三段式启发式

在 100ms 时限内求解，核心是 **模拟退火（SA）+ 贪心修复 + 大邻域搜索（LNS）** 的单线程启发式：

- **信道抽象**：把"（资源块，共享用户集）"抽象为信道，用速率阶梯表 + 撤销日志支持 O(小) 增量评估，
  避免每次改动全量重算。
- **组对齐初始解 + 宏移动**：按 RU/子带结构生成对齐的初始分配，SA 中引入跨子带波束迁移等宏观算子。
- **决赛适配的关键修复**：
  - 重叠边界资源块（同一 res 属两个子带）只调度一次，收尾按所属子带波束能力均值精确重排；
  - 低预算（`beamMaxNum < T`）解锁——允许移走源子带唯一波束，打破"前 beamMaxNum 个子带激活"死锁；
  - 信道激活状态变化强制重评估；P=32 重叠信道等价键修复，避免不同速率信道错误共用速率表；
  - `pairRebalance` 增量化：预计算每用户总贡献，每对资源块只做 O(N) 扣除。
- **等价加速**：`polish` 精修维护每用户累计传输能力，固定迭代下与旧版逐字节一致，纯提速不改语义。
- **工程验证**：g++ 11.4.0 / C++17 / `-O2` 编译通过；三份官方开放样例经独立 Python 计分器全部合法；
  AddressSanitizer + UBSan 边界压测通过；墙钟均值约 88ms、P95 约 89ms、最大 < 91ms，稳定在时限内；
  纯单线程，无多线程 / shell / 网络 / 输入指纹 / 直接映射，通过禁用项静态扫描。

在 `beamMaxNum < T` 的重叠低预算场景，决赛版相对基线平均 **+5.3%**（代表压测例 6672 → 8776 bit）。
